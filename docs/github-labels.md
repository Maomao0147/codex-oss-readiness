# GitHub Labels

This repository keeps a label reference in `.github/labels.yml`. GitHub does not automatically import this file by itself, so create or update labels manually in the GitHub web UI after publishing the repository.

## Manual Setup

1. Open the public repository on GitHub.
2. Go to Issues.
3. Open Labels.
4. Create or edit the labels listed below.
5. Use these labels during issue triage and pull request review.

## Label Set

| Label | Color | Description |
| --- | --- | --- |
| `bug` | `#d73a4a` | Something is broken or confusing. |
| `enhancement` | `#a2eeef` | New or improved behavior. |
| `documentation` | `#0075ca` | Documentation, examples, guides, or templates. |
| `good first issue` | `#7057ff` | Small, clear, beginner-friendly work. |
| `needs info` | `#d876e3` | Waiting for reproduction steps, context, or maintainer clarification. |
| `triaged` | `#0e8a16` | Reviewed by a maintainer and has a clear next step. |
| `release` | `#fbca04` | Release preparation, changelog, or version work. |

## How to Use Labels

- Add one type label: `bug`, `enhancement`, `documentation`, or `release`.
- Add `needs info` only when a specific question is blocking progress.
- Add `triaged` after the issue has acceptance criteria or a clear decision.
- Add `good first issue` only when a new contributor can finish it without broad project knowledge.
