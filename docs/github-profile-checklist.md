# GitHub Profile 设置清单

Codex for OSS 申请会查看 GitHub username 和 public repository。一个清晰可信的 profile 能帮助别人理解你是谁、维护什么、怎样联系你。

## 基础设置

- [ ] 使用稳定头像。
- [ ] 写一句简短 bio，说明你正在维护或学习的开源方向。
- [ ] 添加可公开的联系入口，例如网站、邮箱或社交链接。
- [ ] 检查邮箱隐私设置，必要时使用 GitHub no-reply 邮箱。
- [ ] 确认 profile 中没有不想公开的个人信息。

## 仓库展示

- [ ] Pin `codex-oss-readiness` 或你的真实候选 OSS 仓库。
- [ ] 仓库 About 区域填写 description。
- [ ] 仓库 About 区域添加 topics，例如 `github`, `open-source`, `cli`, `codex`, `readiness`。
- [ ] 仓库 README 第一屏说明项目解决什么问题。
- [ ] 仓库默认分支是 `main`。

## 维护者信号

- [ ] 有清晰 commit 历史。
- [ ] 有 issue 和 pull request 记录。
- [ ] 有 GitHub Actions CI 记录。
- [ ] 有至少一个 release。
- [ ] 有贡献说明和模板，方便别人参与。

## 可选：Profile README

如果你的 GitHub 用户名是 `YOUR_USERNAME`，可以创建一个同名 public 仓库 `YOUR_USERNAME/YOUR_USERNAME`，其中的 README 会显示在 profile 首页。

一个简单模板：

```markdown
# Hi, I am YOUR_NAME

I am learning and practicing open-source maintenance with GitHub.

Current OSS project:

- codex-oss-readiness: a checklist CLI and learning workspace for preparing an OSS repository for Codex for OSS application readiness.

I am practicing:

- issues and pull requests
- CI checks
- releases
- contributor documentation
```
