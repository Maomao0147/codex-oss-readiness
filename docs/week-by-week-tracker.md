# 4 周执行追踪表

这个追踪表把学习、项目建设、维护练习和申请准备串起来。每完成一项，就在 GitHub 上留下对应证据：commit、issue、pull request、release 或文档更新。

## 第 1 周：GitHub 基础

练习入口：先完成 [`first-commit-and-publish.md`](first-commit-and-publish.md)，再完成 [`first-pr-lab.md`](first-pr-lab.md)。

- [ ] 配置本仓库的 Git 姓名和邮箱。
- [ ] 用 `docs/github-skills-matrix.md` 记录本周 GitHub 技能证据。
- [ ] 阅读 `docs/commit-and-branch-guide.md`。
- [ ] 完成第一次 commit。
- [ ] 阅读 `docs/github-auth-and-push-guide.md`。
- [ ] 在 GitHub 创建 public 仓库。
- [ ] 按 `docs/repository-settings-checklist.md` 完成基础设置。
- [ ] 推送 `main` 分支到 GitHub。
- [ ] 创建第一个 issue。
- [ ] 按 `docs/github-labels.md` 创建基础 labels。
- [ ] 按 `docs/issue-triage-guide.md` 给第一个 issue 添加标签。
- [ ] 创建第一个分支。
- [ ] 通过 pull request 合并一次 README 小改动。
- [ ] 按 `docs/pr-review-guide.md` 完成一次 PR review 或 merge note。
- [ ] 查看第一次 GitHub Actions CI 结果。
- [ ] 如果 CI 失败，按 `docs/ci-debugging-lab.md` 完成一次排查记录。

## 第 3 周：真实维护练习

- [ ] 从 `docs/maintenance-backlog.md` 复制至少 3 个 issue 到 GitHub。

- [ ] 每个 issue 用一个分支处理。

- [ ] 准备 `v0.1.0` release notes。
